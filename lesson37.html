Lesson 37 • Physics 356

Convolutional Neural Networks (CNNs)

⏱ ~35 min read

Learning Objectives

  • Explain how convolutional layers exploit spatial structure and compare them to fully-connected layers.
  • Describe the convolution operation mathematically and identify its key hyperparameters (filter size, stride, padding).
  • Explain max pooling and why it provides translation invariance.
  • Identify the components of a standard CNN architecture (Conv → ReLU → Pool → FC → Softmax).
  • Recognize CNNs as a natural extension of the MLPs from Lessons 35–36.

Notation used in this lesson

\(F\)Filter (kernel) size
\(K\)Number of filters (channels out)
\(z_{ij}^{(k)}\)Feature map value at position \((i,j)\), filter \(k\)
\(W_{mn}^{(k)}\)Filter \(k\) weight at position \((m,n)\)
\(p_{ij}\)Pooled output at position \((i,j)\)

🌍 Background & Motivation

The limitation of fully-connected networks on images

The MLPs from Lessons 35–36 treat every input feature identically — a pixel at position (0,0) in an image is treated no differently from one at position (100,100). This ignores the rich spatial structure of image data: nearby pixels are highly correlated, and useful features like edges, corners, and textures are local patterns that appear throughout the image.

A fully-connected layer with \(d = 1024\) inputs and 1024 outputs has \(1024^2 \approx 10^6\) weights. For a 256×256 grayscale image, \(d = 65536\) inputs — the weight count becomes computationally intractable, and the model has no built-in way to share what it learns about an edge in one part of the image with other parts.

Convolutional Neural Networks (CNNs) solve both problems through parameter sharing and local connectivity: the same small filter is applied at every position, dramatically reducing parameters while encoding the spatial inductive bias that local features matter.

Physics applications

CNNs are now standard tools across physics: galaxy morphology classification and gravitational lens detection in astrophysics; phase identification from diffraction patterns in materials science; SAR image interpretation in space domain awareness; and particle track classification in high-energy physics experiments.

🧠 Key Concepts

1. The Convolution Operation

A convolutional layer uses a small filter (kernel) of size \(F \times F\) (e.g., \(3 \times 3\), \(K = 9\) weights) that slides across the input, computing a dot product at each position. The same filter is applied everywhere — this is parameter sharing. The result is a feature map encoding where the filter pattern appears in the input.

\[ z_{ij}^{(k)} = \sum_{m=0}^{F-1}\sum_{n=0}^{F-1} W_{mn}^{(k)} \cdot a_{(i+m)(j+n)} + b^{(k)} \]

Where \(F\) is the filter size, \((i,j)\) is the output position, and \(k\) indexes the filter. A typical convolutional layer uses \(K\) filters in parallel, producing \(K\) feature maps (output channels).

Spatial locality: Each output neuron depends only on a small local region (the receptive field) of the input — exactly right for learning local features like edges and corners.

2. Key Hyperparameters

HyperparameterDescriptionTypical Values
Filter size \(F\)Width/height of the sliding kernel3, 5, 7
Stride \(s\)Step size when sliding the filter1, 2
Padding \(p\)Zeros added around the input border to control output size0 (valid), 1 (same)
Filters \(K\)Number of distinct filters (output channels)32, 64, 128, 256

Output feature map size: \(\left\lfloor \frac{H - F + 2p}{s} \right\rfloor + 1\) (same formula for width).

3. Max Pooling

After a convolutional layer, a pooling layer reduces spatial dimensions. Max pooling takes the maximum value in each non-overlapping region (e.g., \(2 \times 2\)):

\[ p_{ij} = \max_{m,n \in \text{region}} a_{(2i+m)(2j+n)} \]

Pooling provides translation invariance — a feature detected slightly off-center still produces the same output — and reduces computation for subsequent layers by halving spatial dimensions.

4. Typical CNN Architecture

A classic pattern: stack Conv → ReLU → Pool blocks, then flatten and use fully-connected layers for classification:

Input image → [Conv + ReLU] → [MaxPool 2×2] → [Conv + ReLU] → [MaxPool 2×2] → Flatten → FC → Softmax → Class label

Early layers learn low-level features (edges, corners). Deeper layers combine these into higher-level features (shapes, objects). The fully-connected layers at the end perform classification using the learned feature representation.

5. Parameter Count: CNN vs. Fully-Connected

Consider classifying 32×32 grayscale images into 10 classes:

Layer TypeParameters
FC layer (1024 → 1024)\(1024 \times 1024 + 1024 \approx 1.05 \times 10^6\)
Conv layer (3×3, 32 filters, 1 input channel)\(3 \times 3 \times 1 \times 32 + 32 = 320\)
Key insight: A convolutional layer with 32 filters of size 3×3 uses only 320 parameters to process an entire image of any size — compared to millions for a fully-connected layer. This is the power of parameter sharing.

💻 MATLAB Tips & Tricks — Manual 2D Convolution

This example applies a 3×3 Sobel edge-detection filter to a grayscale image by manually sliding the kernel — no toolbox required. It illustrates exactly what a convolutional layer computes. 

%% Lesson 37 — Manual 2D Convolution
% Apply a 3x3 edge-detection filter to a grayscale image

I = double(imread('cameraman.tif'));  % 256x256 grayscale
[H, W] = size(I);

% Sobel filter (detects horizontal edges)
kernel = [-1 -2 -1;
           0  0  0;
           1  2  1];
F = 3;  pad = 1;

% Zero-pad the image
I_pad = padarray(I, [pad pad], 0);

% Slide the kernel across the image (this is what conv2dLayer does internally)
feature_map = zeros(H, W);
for i = 1:H
    for j = 1:W
        patch = I_pad(i:i+F-1, j:j+F-1);
        feature_map(i,j) = sum(sum(kernel .* patch));
    end
end

figure;
subplot(1,2,1); imagesc(I);           colormap gray; title('Original');    axis image off;
subplot(1,2,2); imagesc(feature_map); colormap gray; title('Edge Filter'); axis image off;
MATLAB Deep Learning Toolbox: In practice, use convolution2dLayer, maxPooling2dLayer, and fullyConnectedLayer to build CNNs without manual loops. The toolbox handles backpropagation automatically.

🔗 Looking Ahead — From Spatial to Sequential

CNNs are remarkably effective for data with spatial structure: images, spectrograms, 2D field measurements. But they have a limitation — their receptive field grows slowly with depth, making it hard to capture long-range dependencies. An edge on the left side of an image and a shape on the right are not naturally connected in a CNN.

For sequential data — text, time series, orbital maneuver histories — the key challenge is that the meaning of an element can depend on something arbitrarily far away in the sequence. Lesson 38 introduces the attention mechanism, which allows every position in a sequence to directly attend to every other position in a single operation, regardless of distance.
