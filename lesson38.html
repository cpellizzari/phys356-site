Transformers, LLMs & Diffusion Models

⏱ ~60 min read

Learning Objectives

  • Describe the transformer block (multi-head attention, add & norm, feed-forward) and how stacking blocks gives depth.

  • Identify key physics applications of transformers and diffusion models.

🌍 Background & Motivation

From discriminative to generative models

The networks from Lessons 35–37 are discriminative models — they map inputs to class labels or probabilities. The architectures in this lesson do something more ambitious: transformers process sequences with arbitrary long-range dependencies (making LLMs like ChatGPT possible), and diffusion models learn to generate entirely new data — images, molecules, audio — from noise.

Why this matters for physicists

AlphaFold 2/3 uses transformers to predict protein structure from amino-acid sequences. Diffusion models generate novel drug molecules and reconstruct images from noisy measurements — a problem deeply familiar to physicists (MRI, CT, lidar). Understanding these architectures at a mechanistic level is increasingly essential for anyone working at the frontier of physics and engineering.

🔄 Transformers & the Attention Mechanism

The problem with sequences

For sequential data (text, time series, sensor streams) the central challenge is long-range dependency: the meaning of a word can depend on something said many sentences ago. Recurrent networks (RNNs, LSTMs) struggled because they process tokens step-by-step and compress all past context into a fixed-size hidden state — distant information fades. Transformers solve this with attention: every token can directly look at every other token in one step, regardless of distance.

Step 1 — Word Embeddings: turning tokens into vectors

Neural networks operate on numbers, not words. The first step is to map each word (or sub-word token) to a dense vector in \(\mathbb{R}^d\), called an embedding. These vectors are learned during training, not hand-designed.

A famous property of well-trained embeddings is that arithmetic on the vectors captures semantic relationships:

\[ \text{embed}(\text{"king"}) - \text{embed}(\text{"man"}) + \text{embed}(\text{"woman"}) \;\approx\; \text{embed}(\text{"queen"}) \]

In practice, embedding dimensions are large: GPT-2 uses \(d=768\), GPT-3 uses \(d=12{,}288\). The intuition is that different directions in the embedding space encode different semantic features (gender, animacy, tense, …). You can explore this interactively at projector.tensorflow.org — drag and rotate to see clusters of semantically similar words in 3-D.

Positional encoding: Because attention treats all positions symmetrically, the transformer must be explicitly told the order of tokens. A positional encoding vector — a fixed sinusoidal pattern indexed by position — is added to each embedding before the first attention layer. Without it, "dog bites man" and "man bites dog" would look identical to the network.

Step 2 — Scaled Dot-Product Attention

Let \(\mathbf{x}_i \in \mathbb{R}^d\) be the input embedding vector for token \(i\) — the word embedding plus its positional encoding. Each token is linearly projected into three distinct roles using learned weight matrices \(\mathbf{W}_Q\), \(\mathbf{W}_K\), \(\mathbf{W}_V\):

  • Query \(\mathbf{q}_i = \mathbf{W}_Q \mathbf{x}_i\): the “question” that token \(i\) is asking. It encodes what kind of information token \(i\) needs from the rest of the sequence in order to build its output representation. The index \(i\) marks the token that is doing the asking — every token generates exactly one query.
  • Key \(\mathbf{k}_j = \mathbf{W}_K \mathbf{x}_j\): a compact descriptor of what information token \(j\) contains. Think of it as a label on a file folder that tells you whether this folder is worth opening. The dot product \(\mathbf{q}_i \cdot \mathbf{k}_j\) is a relevance score: how well does token \(j\)’s content match what token \(i\) is looking for? The index \(j\) runs over all tokens being “looked at.” In self-attention, \(j\) ranges over the same sequence as \(i\) — every token can look at every other token, including itself.
  • Value \(\mathbf{v}_j = \mathbf{W}_V \mathbf{x}_j\): the actual information that token \(j\) contributes if attention weight is high. The key decides whether token \(j\) gets selected; the value decides what information gets transferred. Think of it as the contents inside the file folder. The key and value are both indexed by \(j\) because they both belong to the token being attended to.

Vectors \(\mathbf{q}_i, \mathbf{k}_j, \mathbf{v}_j\) vs. matrices \(\mathbf{Q}, \mathbf{K}, \mathbf{V}\). The vector notation gives one token’s projection at a time. In practice we stack all \(n\) tokens into matrices simultaneously: \(\mathbf{Q} \in \mathbb{R}^{n \times d_k}\) (row \(i\) is \(\mathbf{q}_i^\top\)), \(\mathbf{K} \in \mathbb{R}^{n \times d_k}\) (row \(j\) is \(\mathbf{k}_j^\top\)), \(\mathbf{V} \in \mathbb{R}^{n \times d_v}\) (row \(j\) is \(\mathbf{v}_j^\top\)). The matrix product \(\mathbf{Q}\mathbf{K}^\top \in \mathbb{R}^{n \times n}\) computes all \(n^2\) relevance scores in one operation: its \((i,j)\) entry is exactly \(\mathbf{q}_i \cdot \mathbf{k}_j\).

The output for position \(i\) is a weighted average of all values, where weight \(a_{ij}\) encodes how relevant token \(j\) is to token \(i\):

\[ a_{ij} = \frac{\exp\!\left(\mathbf{q}_i \cdot \mathbf{k}_j \,/\, \sqrt{d_k}\right)} {\sum_{l=1}^n \exp\!\left(\mathbf{q}_i \cdot \mathbf{k}_l \,/\, \sqrt{d_k}\right)}, \qquad \mathbf{z}_i = \sum_{j=1}^n a_{ij}\,\mathbf{v}_j \]

In matrix form (all positions at once):

\[ \text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V} \]
Why \(\sqrt{d_k}\)? For large \(d_k\), dot products grow in magnitude, pushing softmax into near-zero gradient regions. Dividing by \(\sqrt{d_k}\) keeps the inputs to softmax in a well-conditioned range.

Concrete example: "cat chases mouse"

In self-attention, every token simultaneously plays both roles: it generates a query (it is asking questions about the other tokens) and it exposes a key and value (it is available to be queried by others). Consider the three-token sentence "cat chases mouse". The attention weight \(a_{ij}\) asks: when building token \(i\)’s output representation, how much should it draw on token \(j\)?

Multi-Head Attention

A single attention operation can only capture one type of relationship at a time. Transformers run \(h\) attention "heads" in parallel, each with independently learned \(\mathbf{W}_Q^{(i)}, \mathbf{W}_K^{(i)}, \mathbf{W}_V^{(i)}\) matrices. The outputs are concatenated and projected:

\[ \text{MultiHead}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\,\mathbf{W}_O \]

Different heads specialize: one head might track grammatical agreement (subject–verb), another might track co-reference (who “they” refers to), another might attend to nearby position. The model learns these specializations automatically from data.

The two sublayers each serve a distinct role. Multi-Head Self-Attention allows every token to gather information from any other position in the sequence — it handles long-range dependencies. The Feed-Forward Network (a two-layer MLP applied independently to each token’s position) adds per-token nonlinear processing and increases representational capacity. Layer Normalization after each Add step stabilizes the magnitude of activations, which is critical for training deep stacks. GPT-2 uses \(N_\ell = 12\) blocks; GPT-3 uses 96. Depth is the primary driver of capability.

Self-attention vs. cross-attention: In self-attention, Q, K, V all come from the same sequence (used in encoders and decoder-only LLMs). In cross-attention (encoder-decoder), Q comes from one sequence (the decoder) and K, V from another (the encoder output) — enabling tasks like translation where two different sequences must communicate.

🤖 Large Language Models (LLMs)

What is an LLM?

A Large Language Model is a transformer trained on massive text corpora (hundreds of billions of tokens) to predict the next token in a sequence. The training task is deceptively simple — predict the next word — but requires the model to develop rich internal representations of language, facts, and reasoning in order to do it well.

\[ p(\text{token}_{t+1} \mid \text{token}_1, \ldots, \text{token}_t) = \text{softmax}\!\left(\mathbf{W}_\text{out}\, \mathbf{h}_t\right) \]

where \(\mathbf{h}_t\) is the final transformer layer’s hidden state at position \(t\). At inference time (using the trained model to generate text), tokens are sampled autoregressively: each predicted token is fed back as the next input to generate the following one, continuing until the model generates an end-of-sequence token.

Training: Pre-training + Fine-tuning

StageTaskDataOutcome
Pre-training Next-token prediction (self-supervised) Entire internet, books, code (∼10T tokens) General language understanding and world knowledge
Supervised Fine-Tuning (SFT) Imitate expert demonstrations Human-written instruction–response pairs Follows instructions reliably
RLHF Maximize human preference reward Human preference rankings of model outputs Aligned, helpful, safe behavior

Scale and emergent abilities

LLM capabilities improve predictably with scale (parameters × data × compute). Certain abilities — multi-step reasoning, code generation, in-context learning — appear to emerge discontinuously as models scale, even though the training task never changes. This emergent behavior is not fully understood and is an active research area.

Physics applications of LLMs and transformers

  • AlphaFold 2/3: transformer + equivariant layers for protein structure prediction from amino-acid sequence.
  • Plasma physics: transformers for time-series modeling of tokamak plasma disruptions.
  • Climate & weather: Pangu-Weather, GraphCast — transformer-based global forecast models.
  • Space domain awareness: sequence modeling of orbital maneuver and conjunction patterns.

🌫️ Diffusion Models

The core idea: learn to denoise

Diffusion models are a class of generative model that learn to generate data by learning to reverse a noising process. The key insight is elegant: if a neural network can learn to remove a small amount of noise from a corrupted image, then iteratively applying that denoiser to pure random noise will gradually produce a coherent image.

How do diffusion models compare to GANs and VAEs?

Diffusion models were not the first approach to generative modeling. Two earlier approaches are worth knowing:

  • Variational Autoencoders (VAEs): An encoder network compresses the input image into a compact latent vector \(\mathbf{z}\), and a decoder reconstructs an image from \(\mathbf{z}\). New images are generated by sampling a random \(\mathbf{z}\) and decoding it. VAEs are stable to train but tend to produce blurry outputs because the loss averages over plausible reconstructions.
  • Generative Adversarial Networks (GANs): Two networks compete — a generator \(G\) produces fake images, and a discriminator \(D\) tries to tell real from fake. The generator learns to fool the discriminator. GANs can produce extremely sharp images but are notoriously unstable to train (mode collapse, training divergence).

Diffusion models have a stable, simple training objective and iteratively refine outputs over many steps. Each denoising step makes a small, learned correction — errors do not compound catastrophically. This stability, combined with their ability to match complex distributions, is why they have largely replaced GANs for high-quality image generation.

Forward process — gradually adding noise

Given a clean data sample \(\mathbf{x}_0\) (e.g., a 28×28 image), the forward process \(q\) defines a Markov chain that gradually adds Gaussian noise over \(T\) steps. The notation \(q\) (rather than \(p\)) follows variational inference conventions: \(q\) is the fixed, non-learned process we define; \(p_\theta\) will be the learned reverse process.

\[ q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) \;=\; \mathcal{N}\!\bigl(\mathbf{x}_t;\; \underbrace{\sqrt{1-\beta_t}\,\mathbf{x}_{t-1}}_{\text{mean}},\; \underbrace{\beta_t\,\mathbf{I}}_{\text{variance}}\bigr) \]

Breaking down the three arguments inside \(\mathcal{N}(\,\cdot\,;\,\cdot\,,\,\cdot\,)\):

  • First argument \(\mathbf{x}_t\): the random variable being distributed (the noisy image at step \(t\)).
  • Second argument \(\sqrt{1-\beta_t}\,\mathbf{x}_{t-1}\): the mean of the distribution. Multiplying by \(\sqrt{1-\beta_t} < 1\) slightly shrinks the previous image toward zero, preventing the signal from growing unboundedly as noise accumulates.
  • Third argument \(\beta_t\,\mathbf{I}\): the covariance matrix. \(\mathbf{I}\) is the identity (each pixel gets independent noise), and \(\beta_t\) sets the noise variance. The schedule \(\beta_1 < \beta_2 < \cdots < \beta_T\) increases slowly (e.g., linearly from \(10^{-4}\) to \(0.02\)), so noise is added a little at a time.

After \(T \approx 1000\) steps, \(\mathbf{x}_T\) is approximately pure Gaussian noise: the original image is completely destroyed.

Closed-form sampling — jumping to any noise level instantly

To train the denoiser, we need many \((\mathbf{x}_0, \mathbf{x}_t)\) pairs at random timesteps \(t\). Naively this would require running the full \(t\)-step chain for every training example — extremely slow. The key mathematical trick is that the forward process has a closed-form marginal: you can jump directly from \(\mathbf{x}_0\) to \(\mathbf{x}_t\) at any step \(t\) without iterating:

\[ \mathbf{x}_t = \sqrt{\bar{\alpha}_t}\;\mathbf{x}_0 \;+\; \sqrt{1 - \bar{\alpha}_t}\;\boldsymbol{\epsilon}, \qquad \boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}) \]

where \(\bar{\alpha}_t = \prod_{s=1}^{t}(1 - \beta_s)\). As \(t\) increases, \(\bar{\alpha}_t \to 0\) and the formula collapses to pure noise. This follows from the fact that applying the forward step repeatedly is equivalent to one larger Gaussian, since Gaussians are closed under convolution.

Why this matters for training: We can instantly construct a noisy sample at any timestep \(t\) by drawing a single noise vector \(\boldsymbol{\epsilon}\) and computing the formula above. This makes training efficient: pick a random \(t\), corrupt \(\mathbf{x}_0\), train the network to predict \(\boldsymbol{\epsilon}\).

Reverse process — the denoising network

The model learns the reverse: given a noisy sample \(\mathbf{x}_t\) at timestep \(t\), predict the noise \(\boldsymbol{\epsilon}\) that was added. This neural network \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\) is trained with a mean-squared error loss:

\[ L = \mathbb{E}_{t,\,\mathbf{x}_0,\,\boldsymbol{\epsilon}}\! \left[\left\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\right\|^2\right] \]

Why predict \(\boldsymbol{\epsilon}\) (the noise) rather than \(\mathbf{x}_0\) directly? Several reasons, shown empirically by Ho et al. (2020):

  • The target \(\boldsymbol{\epsilon}\) is always a standard Gaussian regardless of timestep, giving the network a consistent prediction target.
  • The resulting loss gives equal weight to all noise levels, rather than down-weighting high-noise (early) steps where the image has little signal.
  • Empirically, noise prediction produces sharper, more realistic samples than predicting \(\mathbf{x}_0\) directly.

Inference means using the trained model to generate new images — the "generation" step as opposed to training. During inference, start from \(\mathbf{x}_T \sim \mathcal{N}(\mathbf{0}, \mathbf{I})\) (pure random noise) and iteratively apply the learned denoiser for \(T\) steps to arrive at \(\mathbf{x}_0\) — a brand-new image that was never in the training set.

Conditional generation — how “a blue car” guides the output

Unconditioned diffusion models generate random samples from the data distribution. Real systems like Stable Diffusion or DALL·E generate images that match a text prompt. How?

The most common technique is classifier-free guidance (Ho & Salimans, 2022). The denoising network \(\boldsymbol{\epsilon}_\theta\) is trained in two modes simultaneously: (1) with a conditioning signal \(c\) — typically the CLIP text embedding of the prompt “a blue car” — and (2) with the conditioning replaced by a null token \(\emptyset\) (unconditional mode). During a single training pass, each batch element is randomly conditioned or unconditioned. At inference, the two network outputs are combined as:

\[ \tilde{\boldsymbol{\epsilon}}_\theta(\mathbf{x}_t, c) = (1 + w)\,\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c) \;-\; w\,\boldsymbol{\epsilon}_\theta(\mathbf{x}_t) \]

Breaking down each term:

  • \(\tilde{\boldsymbol{\epsilon}}_\theta(\mathbf{x}_t, c)\): the guided noise estimate used at each inference step in place of the raw network output.
  • \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c)\): the conditional noise estimate — what the network predicts when given both the noisy image \(\mathbf{x}_t\) and the text embedding \(c\) (“a blue car”).
  • \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t)\): the unconditional noise estimate — what the network predicts with no conditioning signal, as if no prompt were provided.
  • \(w \geq 0\): the guidance scale. Common values are \(w = 7\)–\(12\) for strong prompt adherence.

Rewriting the formula makes the geometry clear:

\[ \tilde{\boldsymbol{\epsilon}}_\theta(\mathbf{x}_t, c) = \underbrace{\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c)}_{\text{conditional baseline}} \;+\; w\!\underbrace{\bigl[\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c) - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t)\bigr]}_{\text{guidance direction}} \]

The bracket is the guidance direction — a vector pointing from the unconditioned prediction toward the conditioned prediction. At \(w = 0\), \(\tilde{\boldsymbol{\epsilon}}_\theta = \boldsymbol{\epsilon}_\theta(\mathbf{x}_t, c)\) — the plain conditional prediction with no extra amplification; the prompt is used but not emphasized beyond its trained signal. As \(w\) increases, each denoising step is pushed further in the prompt-consistent direction. At very large \(w\), outputs become over-saturated or lose diversity — so in practice a moderate value like \(w = 7.5\) balances prompt fidelity against image quality. The conditioning signal \(c\) is typically produced by a frozen text encoder (like CLIP), so the same diffusion backbone generates images for any text prompt without retraining.

The denoising network — U-Net architecture

The neural network \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t, t)\) must take in a noisy image and a timestep scalar, and output a noise estimate of the same spatial size. The standard architecture is a U-Net: a CNN with a contracting encoder path and a symmetric expanding decoder path, connected by skip connections that copy feature maps directly from encoder to decoder at the same resolution.

The timestep \(t\) is injected at each resolution level via a sinusoidal embedding (analogous to the transformer’s positional encoding), telling the network how noisy the current input is so it can calibrate the size of the correction to apply. In modern systems (Stable Diffusion 3, DALL·E 3), the U-Net backbone is sometimes replaced by a Diffusion Transformer (DiT) — a pure attention-based architecture applied to flattened image patches, which scales more favorably to very high resolutions.

What diffusion models produce

Stable Diffusion, DALL·E 3, and Midjourney are all diffusion models. Starting from pure Gaussian noise and iteratively applying the learned denoiser for ~20–50 inference steps (using accelerated samplers like DDIM or DPM-Solver), they produce photorealistic images, artwork, molecules, and more.

Representative applications:

  • Text-to-image: Stable Diffusion (open source), DALL·E 3, Midjourney — generate photorealistic or artistic images from any text prompt in seconds.
  • Protein structure: RFDiffusion generates novel protein backbone geometries from noise directly in 3-D coordinate space, enabling de novo protein design.
  • Weather: GenCast (Google DeepMind) generates ensemble weather forecasts via diffusion over atmospheric state variables, capturing forecast uncertainty naturally.

Physics applications of diffusion models

  • Molecular generation: Generating novel drug-like molecules or crystal structures (DiffSBDD, DiffCSP).
  • Inverse problems: Reconstructing clean signals from noisy measurements — directly analogous to MRI, CT, and SAR image reconstruction in physics.
  • Weather & climate: Probabilistic ensemble forecast generation (SEEDS, GenCast).
  • Simulation acceleration: Generating synthetic training data or emulating expensive physics simulations.

💻 MATLAB Demos

Demo 1 — Scaled Dot-Product Attention heatmap

This script computes attention weights for a small example from scratch — no training, no toolboxes beyond core MATLAB. It produces the attention heatmap from the lesson for any sequence of hand-assigned embeddings. Run it, then change the embedding vectors to see how the heatmap changes. 

%% Lesson 38 — Demo 1: Scaled Dot-Product Attention
% Computes and visualizes the attention weight matrix for a 3-token sequence.
% No training required. Illustrates the Q-K dot product and softmax mechanically.
clear; clc; rng(42);

%% 1. Hand-assign word embeddings (d_model = 4)
%    Columns: "cat"  "chases"  "mouse"
%    Features: (animacy, action, subject-role, object-role)
E = [ 0.9   0.3   0.2;    % animacy
      0.1   0.8   0.2;    % action
      0.7   0.4   0.1;    % subject-role
      0.1   0.4   0.8];   % object-role
% E is 4×3: column j = d=4 dim embedding for token j.

words = {'cat','chases','mouse'};
d_k   = 4;   % key/query dimension

%% 2. Projection matrices W_Q, W_K, W_V  (d_k × d_model)
%    Random but fixed — in a real transformer these are learned.
rng(7);
W_Q = randn(d_k, 4) * 0.6;
W_K = randn(d_k, 4) * 0.6;
W_V = randn(d_k, 4) * 0.6;

Q = W_Q * E;   % d_k × n  (each column is the query for one token)
K = W_K * E;
V = W_V * E;

%% 3. Attention scores:  scores(i,j) = q_i · k_j / sqrt(d_k)
scores = (Q' * K) / sqrt(d_k);   % n × n

%% 4. Softmax row-by-row → attention weights
exp_s = exp(scores - max(scores,[],2));   % subtract row max for stability
A     = exp_s ./ sum(exp_s, 2);           % n × n, rows sum to 1

%% 5. Attention output
Z = A * V';   % n × d_k  (each row = context-aware representation)

%% 6. Visualise attention heatmap
figure('Name','Attention Weights','NumberTitle','off');
imagesc(A);
colormap(flipud(gray));   % dark = high attention
clim([0 1]);
colorbar;
set(gca, 'XTick',1:3, 'XTickLabel',words, ...
         'YTick',1:3, 'YTickLabel',words, ...
         'FontSize',12);
xlabel('Key (attended-to token)');
ylabel('Query (attending token)');
title('Scaled Dot-Product Attention Weights', 'FontSize',13);

% Annotate each cell with its weight value
for i = 1:3
    for j = 1:3
        text(j, i, sprintf('%.2f', A(i,j)), ...
             'HorizontalAlignment','center', 'FontSize',11, ...
             'Color', ternary(A(i,j) > 0.45, 'w', 'k'));
    end
end

fprintf('\nAttention weights A (rows sum to 1):\n');
fprintf('%8s  %8s  %8s\n', words{:});
for i = 1:3
    fprintf('%7s: ', words{i});
    fprintf('%8.3f', A(i,:));
    fprintf('\n');
end
fprintf('\nContext-aware output Z (each row = weighted-average of values):\n');
disp(Z);

%% Helper (MATLAB anonymous function equivalent)
function out = ternary(cond, a, b)
    if cond; out = a; else; out = b; end
end

Demo 2 — DDPM forward process on 2-D data

This script visualises the forward noising process on a 2-D Gaussian mixture — a dataset simple enough to plot but rich enough to show the key behavior. No network is trained; the script demonstrates how clean data is gradually destroyed into Gaussian noise according to the closed-form \(\mathbf{x}_t\) formula. Observe that the two clusters become indistinguishable well before \(t = T\). 

%% Lesson 38 — Demo 2: DDPM Forward Process (visualisation only)
% Demonstrates the closed-form forward noising: x_t = sqrt(ᾱ_t)·x_0 + sqrt(1-ᾱ_t)·ε
% No reverse process / training is included here.
clear; clc; rng(42);

%% ── Noise schedule ─────────────────────────────────────────────────
T          = 200;
beta       = linspace(1e-4, 0.02, T);     % DDPM linear schedule
alpha_bar  = cumprod(1 - beta);           % ᾱ_t  (cumulative product)

%% ── Clean data: 2-D Gaussian mixture ──────────────────────────────
n   = 600;
x0  = [randn(2, n/2) + [2; 0], ...    % cluster centered at (+2, 0)
       randn(2, n/2) + [-2; 0]];      % cluster centered at (-2, 0)
% x0 is 2×n: each column is one data point

%% ── Forward process: sample x_t for several timesteps ─────────────
steps_plot = [1, 20, 60, 120, 200];   % which steps to visualise

figure('Name','DDPM Forward Process','NumberTitle','off', ...
       'Position',[100 100 1100 250]);

for k = 1:numel(steps_plot)
    t   = steps_plot(k);
    ab  = alpha_bar(t);

    eps = randn(size(x0));                          % ε ~ N(0,I)
    xt  = sqrt(ab) .* x0 + sqrt(1-ab) .* eps;      % closed-form x_t

    subplot(1, numel(steps_plot), k);
    scatter(xt(1,:), xt(2,:), 8, '.', 'MarkerEdgeColor','#003087');
    axis([-5 5 -5 5]); axis square; box on;
    title(sprintf('t = %d\n\\bar{\\alpha}_t = %.3f', t, ab), 'FontSize',10);
    if k == 1
        ylabel('x_2'); xlabel('x_1');
    end
end
sgtitle( ...
  'DDPM Forward Process — x_t = \surd\bar{\alpha}_t x_0 + \surd(1-\bar{\alpha}_t) \epsilon', ...
  'FontSize', 12);
