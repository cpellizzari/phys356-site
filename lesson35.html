Gradient Descent & Stochastic Gradient Descent

⏱ ~40 min read + interactive demo

Learning Objectives

  • Derive the gradient descent update rule from first principles.
  • Compute gradients of MSE and binary cross-entropy with respect to model parameters.
  • Explain the role of the learning rate and diagnose too-large vs. too-small values.
  • Distinguish batch gradient descent, SGD, and mini-batch SGD.
  • Implement gradient descent to train the logistic regression classifier from Lesson 34.

Notation used in this lesson

\(\boldsymbol{\theta}\)All model parameters \((\mathbf{w}, b)\)
\(L(\boldsymbol{\theta})\)Loss as a function of parameters
\(\nabla_{\boldsymbol{\theta}} L\)Gradient of loss w.r.t. parameters
\(\alpha\)Learning rate (step size)

🌍 Background & Motivation

The optimization problem

In Lesson 34 we defined a loss function \(L(\boldsymbol{\theta})\) that measures how wrong our model is. Now we need to minimize it. In most ML problems, \(L\) is a function of thousands to billions of parameters, and there is no closed-form solution — we must use iterative numerical optimization.

Gradient descent is the backbone of nearly all ML training. The core idea is beautifully simple: always take a small step in the direction that decreases the loss most steeply — the negative gradient direction.

An analogy

Imagine you are lost in a foggy mountain range and want to reach the lowest valley. You cannot see far, but you can feel which direction is locally downhill. Gradient descent is exactly this: at each step, look at the local slope (the gradient) and step downhill. Given enough small steps, you will find a valley (local minimum).

🧠 Key Concepts

1. Gradient Descent Update Rule

Given the current parameters \(\boldsymbol{\theta}^{(t)}\), we update them by stepping opposite the gradient:

\[ \boldsymbol{\theta}^{(t+1)} = \boldsymbol{\theta}^{(t)} - \alpha \, \nabla_{\boldsymbol{\theta}} L\!\left(\boldsymbol{\theta}^{(t)}\right) \]

Where \(\alpha > 0\) is the learning rate — the step size. We repeat this update until the loss stops decreasing (convergence).

Why the negative gradient? The gradient \(\nabla L\) points in the direction of steepest ascent. Subtracting it therefore moves us in the direction of steepest descent.

2. Gradients for the Logistic Regression Loss

For logistic regression with binary cross-entropy, the gradients work out to a remarkably clean form. For a single example \((\mathbf{x}, y)\):

\[ \frac{\partial L}{\partial \mathbf{w}} = (\hat{p} - y)\,\mathbf{x} \qquad \frac{\partial L}{\partial b} = \hat{p} - y \]

For \(N\) training examples (batch gradient descent):

\[ \nabla_\mathbf{w} L = \frac{1}{N} \sum_{i=1}^{N} (\hat{p}_i - y_i)\mathbf{x}_i = \frac{1}{N} \mathbf{X}^\top (\hat{\mathbf{p}} - \mathbf{y}) \qquad \nabla_b L = \frac{1}{N} \sum_{i=1}^N (\hat{p}_i - y_i) \]
Key insight: The gradient \((\hat{p}_i - y_i)\) is simply the prediction error. When the model is wrong, the error is large and the gradient is large — exactly what we want.

3. The Learning Rate — Most Critical Hyperparameter

The learning rate \(\alpha\) controls the step size at each iteration:

  • Too small: training is painfully slow; many iterations to converge.
  • Too large: overshoots the minimum; loss oscillates or diverges.
  • Just right: converges smoothly in a reasonable number of iterations.

There is no formula for the perfect \(\alpha\). Common starting values: \(10^{-3}\) to \(10^{-1}\). Use the interactive demo below to build intuition.

4. Batch vs. SGD vs. Mini-Batch

VariantGradient Computed OverUpdate FrequencyNoise
Batch GD All \(N\) samples Once per epoch Low — exact gradient
SGD 1 sample Every sample High — noisy but fast
Mini-batch SGD \(B\) samples (e.g., \(B=32\)) Every \(B\) samples Moderate — best trade-off

In practice, mini-batch SGD with \(B \in \{32, 64, 128\}\) is standard. The noise in the gradient acts as a regularizer and helps escape sharp local minima.

5. Epoch vs. Iteration

One epoch = one full pass through the training dataset. One iteration = one gradient update (processes \(B\) samples). With \(N=10000\) and \(B=100\), there are 100 iterations per epoch.

💻 Worked Example — Training the Logistic Classifier

This continues directly from Lesson 34. We implement gradient descent to train the logistic regression classifier on the space-object dataset: 

%% Lesson 35 — Gradient Descent for Logistic Regression
% (Continues from Lesson 34 setup)
rng(1); N = 200;
X0 = [randn(N/2,1)*0.5+1, randn(N/2,1)*0.5+3];
X1 = [randn(N/2,1)*0.5+3, randn(N/2,1)*0.5+1];
X  = [X0; X1];
y  = [zeros(N/2,1); ones(N/2,1)];

sigmoid = @(s) 1 ./ (1 + exp(-s));

%% --- Hyperparameters ---
alpha  = 0.5;      % learning rate
n_iter = 500;      % number of gradient descent steps

%% --- Initialize parameters ---
w = zeros(2,1);    % weight vector
b = 0;             % bias

%% --- Training loop ---
loss_history = zeros(n_iter, 1);

for t = 1:n_iter
    % Forward pass
    s     = X * w + b;            % N×1 raw scores
    p_hat = sigmoid(s);            % N×1 predicted probabilities

    % Loss (cross-entropy)
    eps = 1e-8;
    loss_history(t) = -mean(y .* log(p_hat+eps) + (1-y) .* log(1-p_hat+eps));

    % Gradients (vectorized)
    delta = p_hat - y;             % N×1 prediction errors
    dw    = (X' * delta) / N;      % 2×1
    db    = mean(delta);           % scalar

    % Parameter update
    w = w - alpha * dw;
    b = b - alpha * db;
end

%% --- Final evaluation ---
p_hat_final = sigmoid(X * w + b);
y_pred = double(p_hat_final >= 0.5);
acc = mean(y_pred == y);
fprintf('Final accuracy: %.2f%%\n', acc*100);
fprintf('Final loss: %.4f\n', loss_history(end));
fprintf('Learned weights: w = [%.3f, %.3f], b = %.3f\n', w(1), w(2), b);

%% --- Plot 1: Loss curve ---
figure;
plot(1:n_iter, loss_history, 'b-', 'LineWidth', 2);
xlabel('Iteration'); ylabel('Cross-Entropy Loss');
title(sprintf('Training Loss (lr=%.2f)', alpha));
grid on;

%% --- Plot 2: Decision boundary ---
figure; hold on;
scatter(X0(:,1), X0(:,2), 30, 'r', 'filled', 'DisplayName','Debris (y=0)');
scatter(X1(:,1), X1(:,2), 30, 'b', 'filled', 'DisplayName','Satellite (y=1)');
x1r = linspace(0, 5, 100);
x2b = -(w(1)*x1r + b) / w(2);
plot(x1r, x2b, 'k-', 'LineWidth', 2.5, 'DisplayName','Learned Boundary');
xlabel('Radar Cross Section (norm.)');
ylabel('Brightness Magnitude (norm.)');
title('Trained Linear Classifier'); legend; grid on;
xlim([0 5]); ylim([0 5]);
Experiment: Change alpha to 0.01 and 2.0. Plot the loss curves side by side. What do you observe?
