Neural Networks — Architecture & Forward Pass

⏱ ~45 min read

Learning Objectives

  • Describe a feedforward neural network in terms of layers, neurons, weights, biases, and activations.
  • Write the forward pass equations for a multi-layer network in both scalar and matrix form.
  • Compare common activation functions: sigmoid, tanh, and ReLU, and explain why ReLU is preferred.
  • State the Universal Approximation Theorem and understand its significance and limitations.
  • Implement the forward pass of a shallow (1 hidden layer) neural network in MATLAB.

Notation used in this lesson

\(L\)Number of layers (not loss here)
\(\mathbf{W}^{[\ell]}\)Weight matrix, layer \(\ell\)
\(\mathbf{b}^{[\ell]}\)Bias vector, layer \(\ell\)
\(\mathbf{z}^{[\ell]}\)Pre-activation (linear combination)
\(\mathbf{a}^{[\ell]}\)Activation (post-activation)
\(n^{[\ell]}\)Number of neurons in layer \(\ell\)
\(\phi(\cdot)\)Activation function
\(\hat{\mathbf{y}}\)Network output (predictions)

🌍 Background & Motivation

The limitation of linear models

The linear classifier from Lesson 34 is powerful but fundamentally limited: it can only learn linear decision boundaries. Real data is rarely linearly separable. Satellite orbit families, material phases, particle signatures — these require curved, complex boundaries.

Neural networks overcome this by stacking multiple layers of linear transformations, each followed by a nonlinear activation function. The composition of these layers can represent arbitrarily complex functions — a result formalized by the Universal Approximation Theorem.

Biological inspiration (and where it ends)

Artificial neurons are loosely inspired by biological neurons: they receive inputs, compute a weighted sum, and fire (activate) if the sum exceeds a threshold. However, modern neural networks are best thought of as parameterized function approximators, not brain simulations. The biological analogy is more historical than mechanistic.

🧠 Key Concepts

1. Network Architecture

A feedforward neural network (also called a multi-layer perceptron, MLP) consists of:

  • Input layer: receives the feature vector \(\mathbf{x} \in \mathbb{R}^d\). No computation here.
  • Hidden layers: one or more layers where learning happens. Each neuron receives all outputs from the previous layer.
  • Output layer: produces the prediction \(\hat{\mathbf{y}}\). For regression: linear output. For classification: sigmoid (binary) or softmax (multi-class).

We describe the architecture by listing the number of neurons per layer. Example: a network with input \(d=10\), one hidden layer of 64 neurons, and output of 1 is written as [10 → 64 → 1].

2. The Forward Pass (Single Layer)

For layer \(\ell\), the computation is:

\[ \underbrace{\mathbf{z}^{[\ell]}}_{\text{pre-activation}} = \mathbf{W}^{[\ell]} \mathbf{a}^{[\ell-1]} + \mathbf{b}^{[\ell]} \qquad \longrightarrow \qquad \underbrace{\mathbf{a}^{[\ell]}}_{\text{activation}} = \phi\!\left(\mathbf{z}^{[\ell]}\right) \]

Where \(\mathbf{W}^{[\ell]} \in \mathbb{R}^{n^{[\ell]} \times n^{[\ell-1]}}\) is the weight matrix, \(\mathbf{b}^{[\ell]} \in \mathbb{R}^{n^{[\ell]}}\) is the bias vector, and \(\phi\) is applied element-wise.

3. Full Forward Pass (Two-Layer Network)

A network with one hidden layer and a binary classification output:

\[ \mathbf{a}^{[0]} = \mathbf{x} \quad \text{(input)} \] \[ \mathbf{z}^{[1]} = \mathbf{W}^{[1]} \mathbf{x} + \mathbf{b}^{[1]}, \qquad \mathbf{a}^{[1]} = \text{ReLU}(\mathbf{z}^{[1]}) \] \[ z^{[2]} = \mathbf{W}^{[2]} \mathbf{a}^{[1]} + b^{[2]}, \qquad \hat{y} = \sigma(z^{[2]}) \]

The network applies a learned nonlinear transformation from \(\mathbf{x}\) to \(\hat{y}\) in three matrix operations. The power comes from learning \(\mathbf{W}^{[1]}, \mathbf{b}^{[1]}, \mathbf{W}^{[2]}, b^{[2]}\) jointly.

4. Activation Functions

Without activations, stacking linear layers is useless — you just get another linear layer. Nonlinearity is essential.

FunctionFormulaRangeCommon Use
Sigmoid \(\sigma(z) = \frac{1}{1+e^{-z}}\) \((0,1)\) Binary output layer
Tanh \(\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}\) \((-1,1)\) Hidden layers (older)
ReLU \(\max(0, z)\) \([0,\infty)\) Hidden layers (modern default)
Softmax \(\frac{e^{z_k}}{\sum_j e^{z_j}}\) \((0,1)\), sums to 1 Multi-class output layer
Why ReLU? Sigmoid and tanh saturate (gradient ≈ 0) for large inputs — the "vanishing gradient" problem that makes deep networks hard to train. ReLU has gradient exactly 1 for positive inputs, enabling training of much deeper networks.

5. Parameter Count

For a network \(d \to n_1 \to n_2 \to n_\text{out}\), the total parameters are:

\[ \text{Total params} = \underbrace{d \cdot n_1 + n_1}_{\mathbf{W}^{[1]},\mathbf{b}^{[1]}} + \underbrace{n_1 \cdot n_2 + n_2}_{\mathbf{W}^{[2]},\mathbf{b}^{[2]}} + \underbrace{n_2 \cdot n_\text{out} + n_\text{out}}_{\mathbf{W}^{[3]},\mathbf{b}^{[3]}} \]

Modern networks have millions to billions of parameters. Understanding the count helps you reason about model capacity and overfitting risk.

6. Universal Approximation Theorem

A feedforward network with one hidden layer and a nonlinear activation can approximate any continuous function on a compact domain to arbitrary accuracy, given enough neurons. This is a theoretical existence result — it does not say learning is easy, only that the capacity is there.

Important caveat: The theorem guarantees expressiveness, not learnability. In practice, deep networks (many layers, fewer neurons each) generalize better and are easier to train than very wide shallow networks.

💻 Worked Example — MATLAB Forward Pass

We implement the forward pass of a 2-layer network on the sat_data.mat satellite dataset. This is the setup you will use in HW 35 to build a full training loop. 

%% Lesson 35 — Neural Network Forward Pass
% Dataset: sat_data.mat
% Features: orbital parameters of satellites
% Labels: maneuver type classification

% Load data (assumes sat_data.mat is in the working directory)
load('sat_data.mat');   % provides: X_train, y_train, X_test, y_test
% X_train: N_train × d  (normalized features)
% y_train: N_train × 1  (binary labels: 0 or 1)

[N, d] = size(X_train);
fprintf('Training samples: %d,  Features: %d\n', N, d);

%% --- Network architecture: d → 32 → 16 → 1 ---
n1 = 32;   % hidden layer 1 size
n2 = 16;   % hidden layer 2 size

% Initialize weights (Xavier initialization)
W1 = randn(n1, d)  * sqrt(2/d);     % n1 × d
b1 = zeros(n1, 1);                   % n1 × 1
W2 = randn(n2, n1) * sqrt(2/n1);    % n2 × n1
b2 = zeros(n2, 1);                   % n2 × 1
W3 = randn(1,  n2) * sqrt(2/n2);    % 1  × n2
b3 = 0;                              % scalar

%% --- Activation functions ---
relu    = @(z) max(0, z);
sigmoid = @(z) 1 ./ (1 + exp(-z));

%% --- Forward pass (single sample) ---
x = X_train(1,:)';   % d × 1 — first training example

z1 = W1 * x  + b1;   % n1 × 1
a1 = relu(z1);        % n1 × 1

z2 = W2 * a1 + b2;   % n2 × 1
a2 = relu(z2);        % n2 × 1

z3 = W3 * a2 + b3;   % scalar
y_hat = sigmoid(z3); % predicted probability ∈ (0,1)

fprintf('True label: %d,  Predicted probability: %.4f\n', y_train(1), y_hat);

%% --- Vectorized forward pass (all N training samples) ---
% Transpose X_train to d × N for matrix ops
Xt = X_train';        % d × N

Z1 = W1 * Xt + b1;   % n1 × N
A1 = relu(Z1);        % n1 × N

Z2 = W2 * A1 + b2;   % n2 × N
A2 = relu(Z2);        % n2 × N

Z3 = W3 * A2 + b3;   % 1 × N
P_hat = sigmoid(Z3)'; % N × 1 — predicted probabilities

%% --- Initial loss (before training) ---
eps = 1e-8;
y = y_train;
L = -mean(y .* log(P_hat+eps) + (1-y) .* log(1-P_hat+eps));
acc_init = mean(double(P_hat >= 0.5) == y);
fprintf('Initial loss: %.4f,  Accuracy: %.2f%%\n', L, acc_init*100);

%% NOTE: This is only the forward pass.
%% In HW35 you will implement backpropagation to compute
%% gradients and train the network with SGD.
Xavier initialization: Initializing weights with \(\mathcal{N}(0, \sqrt{2/n_\text{in}})\) keeps activations from exploding or vanishing at initialization. This is a standard best practice.
