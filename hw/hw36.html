HW 36 — Training Parameter Exploration

📘 Related: Lesson 36 🛠 MATLAB + Deep Learning Toolbox required 📁 Dataset: MATLAB DigitDataset

🌍 Background

In the Lesson 36 worked example you trained a small fully-connected network on handwritten digits and observed an underfitting pattern in the Training Progress plot: both training and validation accuracy remained low because 10 hidden neurons are insufficient to recognize 10 distinct digit classes from 784 pixel features. In this assignment you will systematically tune the network to improve performance, and in doing so, identify and diagnose each of the training curve patterns discussed in the lesson.

Use the Deep Learning Toolbox layer API (featureInputLayer, fullyConnectedLayer, trainingOptions, trainNetwork) as shown in the Lesson 36 example. Set rng(356) at the top of your script so your data split is reproducible.

📁 Dataset Setup

Load the data using the same pattern as the Lesson 36 example (200 images per class, 2000 total). The 80/20 train/test split gives you 1,600 training samples and 400 test samples. 

digitPath = fullfile(matlabroot,'toolbox','nnet','nndemos', ...
                     'nndatasets','DigitDataset');
imds = imageDatastore(digitPath, ...
    'IncludeSubfolders', true, 'LabelSource', 'foldernames');
rng(356);
imds_sub = splitEachLabel(imds, 200, 'randomize');

% 80/20 train/test split
[imds_train, imds_test] = splitEachLabel(imds_sub, 0.8, 'randomize');

% Flatten: each 28×28 image → 784-element row vector; N×784 matrix
imgs_train = readall(imds_train);
v_train    = cellfun(@(I) double(I(:))/255, imgs_train, 'UniformOutput', false);
X_train    = [v_train{:}]';          % 1600 × 784

imgs_test  = readall(imds_test);
v_test     = cellfun(@(I) double(I(:))/255, imgs_test,  'UniformOutput', false);
X_test     = [v_test{:}]';           % 400 × 784

Y_train = imds_train.Labels;         % categorical, 1600×1
Y_test  = imds_test.Labels;          % categorical, 400×1

Problem 1 — Fixing Underfitting: Network Size

  1. Train networks with at least four different hidden layer sizes (10, 50, 100, 200 neurons in fc1). Use adam as your optimizer and 50 max epochs. A reusable template: 
    N = 50;   % hidden layer size — vary this
layers = [
    featureInputLayer(784)
    fullyConnectedLayer(N)
    reluLayer
    fullyConnectedLayer(10)
    softmaxLayer
    classificationLayer
];
opts = trainingOptions('adam', 'MaxEpochs', 50, ...
    'ValidationData', {X_test, Y_test}, ...
    'Plots', 'training-progress', 'Verbose', false);
net    = trainNetwork(X_train, Y_train, layers, opts);
Y_pred = classify(net, X_test);
acc    = 100 * mean(Y_pred == Y_test);
fprintf('N=%d  Test acc: %.1f%%\n', N, acc);
  2. For each architecture, classify the Training Progress pattern you observe as underfitting, good generalization, or overfitting. As the network grows, at what size does the pattern transition from underfitting to good generalization?
  3. Present your results in a table (hidden neurons | accuracy | pattern) and identify the architecture you would recommend. Justify your choice.

Problem 2 — Inducing Overfitting

  1. Using a very large network on limited data is a reliable way to produce overfitting. Create a small subset (30 images per class, 300 total) and train a large network on it: 
    rng(356);
imds_small = splitEachLabel(imds, 30, 'randomize');
[imds_s_train, imds_s_test] = splitEachLabel(imds_small, 0.8, 'randomize');

% Flatten (same pattern as Dataset Setup)
imgs_s = readall(imds_s_train);
v_s    = cellfun(@(I) double(I(:))/255, imgs_s, 'UniformOutput', false);
X_s_train = [v_s{:}]';
imgs_st = readall(imds_s_test);
v_st    = cellfun(@(I) double(I(:))/255, imgs_st, 'UniformOutput', false);
X_s_test  = [v_st{:}]';
Y_s_train = imds_s_train.Labels;
Y_s_test  = imds_s_test.Labels;

layers_big = [
    featureInputLayer(784)
    fullyConnectedLayer(200)
    reluLayer
    fullyConnectedLayer(10)
    softmaxLayer
    classificationLayer
];

% Run 1: early stopping active (stop after 6 non-improving validations)
opts1 = trainingOptions('adam', 'MaxEpochs', 200, ...
    'ValidationData',    {X_s_test, Y_s_test}, ...
    'ValidationPatience', 6, ...
    'Plots', 'training-progress', 'Verbose', false);
net1 = trainNetwork(X_s_train, Y_s_train, layers_big, opts1);

% Run 2: early stopping disabled
opts2 = trainingOptions('adam', 'MaxEpochs', 200, ...
    'ValidationData',    {X_s_test, Y_s_test}, ...
    'ValidationPatience', Inf, ...
    'Plots', 'training-progress', 'Verbose', false);
net2 = trainNetwork(X_s_train, Y_s_train, layers_big, opts2);
  2. Examine the Training Progress plot for Run 1. Do you see the overfitting pattern (validation accuracy falls after a peak)? Approximately which epoch does validation accuracy reach its maximum?
  3. Compare the final test accuracy of Run 1 (early stopping) vs. Run 2 (no early stopping). Does early stopping help? Why?

Problem 3 — Effect of Learning Rate

  1. Return to the full 200-per-class dataset. Using your best architecture from Problem 1, use SGD with momentum ('sgdm') and try at least four learning rates: 
    lr = 0.01;   % vary: try 0.001, 0.01, 0.1, 0.5
opts = trainingOptions('sgdm', ...
    'MaxEpochs',        100, ...
    'MiniBatchSize',    64, ...
    'InitialLearnRate', lr, ...
    'Momentum',         0.9, ...
    'ValidationData',   {X_test, Y_test}, ...
    'Plots',            'training-progress', ...
    'Verbose',          false);
net    = trainNetwork(X_train, Y_train, layers, opts);
Y_pred = classify(net, X_test);
acc    = 100 * mean(Y_pred == Y_test);
fprintf('lr=%.3f  Test acc: %.1f%%\n', lr, acc);
    For each learning rate, record the test accuracy and describe the shape of the Training Progress curve.
  2. For the learning rate that gave the best accuracy, include the Training Progress plot in your submission. Which of the four curve patterns from Figure 2 of the lesson does it most resemble?
  3. Now retrain with 'adam' (keeping all other options the same, and removing the explicit InitialLearnRate so it uses the Adam default). Compare its test accuracy and convergence speed to your best 'sgdm' result. Explain in 2–3 sentences why an adaptive optimizer like Adam typically outperforms fixed-learning-rate SGD.
