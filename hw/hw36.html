HW 36 — Training Parameter Exploration

📘 Related: Lesson 36 🛠 MATLAB + Deep Learning Toolbox required 📁 Dataset: MATLAB DigitDataset

🌍 Background

In the Lesson 36 worked example you trained a patternnet(10) on handwritten digits and observed an underfitting pattern in the Performance plot: both training and validation losses remained high because 10 hidden neurons are insufficient to recognize 10 distinct digit classes from 784 pixel features. In this assignment you will systematically tune the network to improve performance, and in doing so, identify and diagnose each of the training curve patterns discussed in the lesson.

Use the same data loading code from the Lesson 36 example. Set rng(356) at the top of your script so your data split is reproducible.

📁 Dataset Setup

Load the data exactly as in the Lesson 36 example (200 images per class, 2000 total, flattened to 784-dim vectors): 

digitPath = fullfile(matlabroot,'toolbox','nnet','nndemos', ...
                     'nndatasets','DigitDataset');
imds = imageDatastore(digitPath, ...
    'IncludeSubfolders', true, 'LabelSource', 'foldernames');
rng(356);
imds_sub  = splitEachLabel(imds, 200, 'randomize');
imgs      = readall(imds_sub);
X         = cell2mat(cellfun(@(I) double(I(:))/255, imgs, ...
                    'UniformOutput', false)')';   % 784 x 2000
classNums = grp2idx(imds_sub.Labels);             % 2000x1
T         = full(ind2vec(classNums'));             % 10 x 2000

Problem 1 — Fixing Underfitting: Network Size

  1. Train networks with at least four different hidden layer sizes: patternnet(10), patternnet(50), patternnet(100), and patternnet(200). For each, record the training accuracy returned by: 
    pred = vec2ind(net(X));
acc  = 100 * mean(pred == classNums');
  2. For each architecture, open the Performance plot and classify the pattern you observe as underfitting, good generalization, or overfitting. As the network grows, at what size does the pattern transition from underfitting to good generalization?
  3. Present your results in a table (hidden neurons | accuracy | pattern) and identify the architecture you would recommend. Justify your choice.

Problem 2 — Inducing Overfitting

  1. Using a very large network on limited data is a reliable way to produce overfitting. Reduce your training subset to 30 images per class (300 total): 
    rng(356);
imds_small = splitEachLabel(imds, 30, 'randomize');
    Flatten and prepare this smaller dataset exactly as before, then train a patternnet(200) on it.
  2. Open the Performance plot. Do you see the overfitting pattern (validation loss rises after a minimum)? Approximately which epoch does the validation loss reach its minimum?
  3. Now retrain with net.trainParam.max_fail = 50 (effectively disabling early stopping) and observe how the final accuracy compares to the run where early stopping was active. Does early stopping help? Why?

Problem 3 — Effect of Learning Rate

  1. Return to the full 200-per-class dataset. Using your best architecture from Problem 1, switch to gradient descent with momentum and try at least four learning rates: 
    net.trainFcn          = 'traingdm';
net.trainParam.lr     = ???;       % try 0.001, 0.01, 0.1, 0.5
net.trainParam.epochs = 300;
net.trainParam.showWindow = false; % suppress GUI in loop
    For each, record the training accuracy.
  2. Re-enable the GUI (showWindow = true) for the learning rate that gave the best accuracy and include the Performance plot in your submission. Which of the four curve patterns from Figure 2 of the lesson does it most resemble?
  3. Compare your best traingdm result to the default trainscg. Which converges to higher accuracy? Which requires fewer epochs? Explain in 2–3 sentences why trainscg typically outperforms a fixed-learning-rate method on small datasets.

Problem 4 — Putting It Together

  1. Based on everything you have explored, identify the single best configuration you found (architecture + training algorithm + any parameter choices). Report its training accuracy.
  2. In the Lesson 36 example we flattened each 28×28 image into a 784-element vector, discarding spatial information. List two specific limitations this imposes on the network's ability to recognize digits, and explain how a Convolutional Neural Network (which you will build in Lesson 37) addresses each limitation.
